# v0

## ~~JSON Schema spec correctness fixes~~ ✓ Complete

> Completed in
> [`../plans/completed/plan_json_schema_correctness.md`](../plans/completed/plan_json_schema_correctness.md).

Four keyword implementations contradicted the JSON Schema Validation 2020-12
spec. All four have been fixed:

- `pattern` is no longer anchored — `hasMatch()` replaces the full-string check
- `integer` accepts whole-number doubles (e.g. `1.0`), non-finite values rejected
- `type` array form (`["string","null"]`) is now parsed and validated correctly
- `uri` and `urn` format validators have been corrected (were swapped)

## ~~JSON Schema: wire existing validators into the rule layer~~ ✓ Complete

> Completed in
> [`../plans/completed/plan_json_schema_wire_validators.md`](../plans/completed/plan_json_schema_wire_validators.md).

Six keywords that had working `Validator` implementations in `validation.dart`
but were unreachable via `SchemaParser` or `JsonSchemaValidator` are now fully
wired in:

`const`, `multipleOf`, `uniqueItems`, `minProperties`, `maxProperties`,
`dependentRequired`

## ~~JSON Schema: missing keywords~~ ✓ Complete

> Completed in
> [`../plans/completed/plan_json_schema_missing_keywords.md`](../plans/completed/plan_json_schema_missing_keywords.md).

Structural keywords that were absent entirely and whose omission caused invalid
documents to silently pass validation — all four groups are now implemented:

- `contains` / `minContains` / `maxContains`
- `prefixItems` (and correct 2020-12 semantics for `items` with a prefix, including boolean `items: false`)
- `patternProperties`
- `additionalProperties` as a sub-schema (previously only `false` was handled; guard removed)

## JSON Schema: standard format strings

> Investigated in
> [`../plans/plan_json_schema_formats.md`](../plans/plan_json_schema_formats.md).

Eleven format strings defined by the spec (§7.3) are not registered in
`StringFormatValidator`. Schemas using these formats silently produce no
violations today:

`idn-email`, `hostname`, `idn-hostname`, `ipv4`, `ipv6`, `uri-reference`,
`iri`, `iri-reference`, `uri-template`, `json-pointer`, `relative-json-pointer`
